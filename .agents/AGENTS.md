# logseq-to-obsidian Workspace Rules

## Project Structure
- `src/`: Core codebase containing converter packages
- `tests/`: Integration and unit tests

## Active Technologies & Tooling
- Python 3.14+
- `uv` for dependency management and running commands
- `pytest` for unit and integration testing
- `ruff` for code style checks and formatting

## Code Style & Conventions
- Python 3.14+: Follow standard Python coding conventions.
- Ruff line-length limit is set to 120 (configured in `pyproject.toml`).
